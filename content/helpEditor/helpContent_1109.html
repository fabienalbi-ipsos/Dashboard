Overview

The underlying principle of the BI Business Dashboard is that it is a role based Dashboard with related access, so not all users are eligible to the Business Dashboard; only selected roles.

A list of Roles has been identified, and each role provides a different version of the Dashboard.

Access is granted based on Tree Nodes in the 3 BI Trees. Access to each tree node will determine the linked report or reports available in each Dashboard â€“ these will be standard & will differ depending on the tree node.

A role does not correspond only to one tree node but can correspond to multiple ones, but a tree node can only correspond to one role.

Each user is setup for a defined list of roles and/or associated scopes

As some users could have multiple roles & scopes associated, a default setup with a default role and a default scope for each role is defined in the user profile table. The default scope is a tree node in one of the three trees.

The Dashboard is located on a dedicated BI tab: a user on this tab will/can have a different security in default versus the regular BI access on other tabs.

Some BI automated e-mail alerts are triggered using this same approach.

Since BI trees are maintained & updated significantly at least once per year, a process of disabling the access to the Dashboard & related alerts is in place to avoid users accessing inaccurate information while the update in is progress

Login directly to the dashboard is also possible by navigating directly to bidashboard.ipsos.


Navigation

Home / Main Menu


There are seven user controls on the BI Business Dashboard. You can change the content of each of these controls by clicking on the control. This will open a menu which allows you to change values.

Whenever you start the dashboard, default values are populated for each control. This makes it easy to launch the dashboard without the need to make any selections at all, since each control will already contain a value.

Changing any value in any control will automatically trigger a refresh of the dashboard page; all data on the page will be updated based on the change you made to a control.

These are the controls available to you

  1. Role
  2. Scope
  3. Year
  4. Month
  5. Currency
  6. Exchange Rate Type
  7. Rounding 


Role

Most users only have one role, so there will likely not be any options available to most users. Your default Role is assigned and saved in your user profile record. You can request to change your default role if you wish. Whenever you start a new session in the Dashboard, the page will load with your default Role. During a session, you can select a different role, if you have multiple roles.


Scope

This control will show one or more trees with all available business units that you can choose. You can select any lower level of a tree, down to the lowest level, which is the Management Business Unit. You cannot select a tree level higher than you have been granted by the security team. 


Your default Scope has been assigned and saved in your user profile record. You can request a change to your default Scope if you wish. Whenever you start a new session in the Dashboard, the page will load with your default Scope. During a session, you can select a different Scope by clicking the Scope control. When you log out or close your browser, the next time you log in, you will automatically load the dashboard with your default Scope.




If your scope includes options on multiple trees, you will see a button for each tree. You can toggle from one tree to another by clicking a tree button.

You can expand or collapse a tree using the +/- buttons on each tree node. 

Select a tree node by clicking the checkbox next to the tree node name. This will trigger a refresh of the page, and show an updated dashboard for the selected tree node.


Year

The application supplies the default year based on the current audit year. You can change this value to a previous year, but data for the dashboard didn't not start until 2016, so there is no data to report prior to this time.

Month

The application supplies the default month based on where we are in the reporting cycle relative to your current session date. The calendar of reporting period cutoff dates is managed by the Group Controlling Team. Typically the month will change to the next month around the 7th of each month. During a session, you can change the displayed month to any value from Jan to Dec. Next time you log in, the default value will be displayed.

Currency

Your user profile record contains a default reporting currency. You can request a change to the default value if you wish. Your default currency will be used each time you log into the Dashboard. You can select any other available currency during your session. Note that the list of currencies only shows those currencies which are appropriate for you, based on your scope. For example, you cannot select Chilean Pesos if your scope is Public Affairs Canada. 

All users have access to the EURO currency as an option, plus all currencies in your scope.

Exchange Rate Type

The application default Rate Type is Prior Full Year Average. You can select another Rate Type from the Drop down list.

Rounding

The application default rounding option for most users is Thousands. For some users the default rounding option is Millions. 

Millions rounding is the default for users in countries with very small currencies, relative to the EUR, or for users with Global Scope.

Mobile Devices

Home / Main Menu

To access your dashboard on a mobile device, you must be connected to the Ipsos wireless network. You cannot access the dashboard from mobile network or wireless network outside of the Ipsos firewall.

While we have tested the dashboards on many different devices, we cannot guarantee that it will work on every mobile device, given the large number of different devices and operating systems.

We have tested and confirm that the following devices will work well

  • Ipad
  • Iphone 5+
  • Galaxy 5+


On mobile devices, the top navigation bar is hidden inside a menu button (the hamburger button) on the top right corner of your screen, and is displayed when you tap the button.

Open any section of the dashboard by tapping the down arrow on a section header.



When the menu is open, make a selection from one of the controls and the application will refresh your dashboard accordingly.

Some KPI content is not available on mobile, due to the smaller screen size. For example on a phone, not all columns of the P&L are displayed; only the three most relevant columns are shown. 

Also, linked reports are not available on mobile, again due to screen size. 

To take advantage of full functionality, please log in via your laptop.


Headlines

Home / Main Menu

The Headlines section shows a summary of six key KPI values; each one is a clickable link that takes you directly to the full KPI content further down in the dashboard. On the right, you see the top five proposals won during the displayed month. 

All content is displayed based on the displayed controls at the top of the page. 

Top five Proposals won shows the amounts in the displayed currency, rounded to thousands, regardless of the rounding option selected. In other words, if you have chosen to round the dashboard in Millions, the values of the top five proposals won is displayed rounded to thousands. This is done since each row of the data represent a single proposal; rounding to millions would result in very small fractional numbers in most cases.

Pipeline

Pipeline content is a snapshot, finalized on the last day of the calendar month. 

The section title bar shows the total value of the year to date pipeline, plus a percentage change versus last year to date.  

The details show a table with columns for the Current Year to date, Last Year to Date, and a variance between these two values. 

Rows:

Proposal Count = the quantity of pending proposals in the pipeline

Proposal Value = the value of the pending proposals in the currency selected.

Probability % = the average probability percentage of all proposals in the selected scope

Pipeline Value = Proposal Value multiplied by Probability %

Proposal Average Value = Proposal Value divided by Proposal Count. Amount is displayed in the selected currency and rounding option.



Linked Reports


The list of linked reports is displayed in the drop down list. Click on one of the reports listed to generate the report in a new tab. All reports listed are available directly in BI as well.


The reports available are based on the Role and Scope of the user

  • Pipeline by Management BU
    • Detail Level 3 Analysis Type: Summary
    • Available to all roles
  • Pipeline by Management Country/Cluster
    • Detail Level 2 Analysis Type: Summary
    • Available to all roles with access to tree level 5 or above
  • Pipeline by Management Country
    • Detail Level 2 Analysis Type: Summary
    • Available to all roles with access to tree level 5 or above
  • Pipeline by Parent Client
    • Detail Level: No SubTotal, Analysis Type: Parent Client


Alerts

Alerts are displayed in the top right corner of the dashboard. Access them by clicking the bell icon. The number of alerts is shown in a circle above the bell icon.

There are three alerts:

  • Pending Proposals due this month
  • Jobs with negative OPBBGC %
  • Jobs with significant variances on Professional Time

Alerts are only displayed for the current calendar month. If you select a period other than the current calendar month, no alerts are shown.
